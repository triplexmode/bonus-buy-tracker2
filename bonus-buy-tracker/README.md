# 🎰 Bonus Buy Tracker

Стриминговый виджет и админка для отслеживания бонус-баев в казино/слотах.

## Страницы

| URL | Назначение |
|-----|-----------|
| `/admin.html` | Админ-панель для управления данными |
| `/widget.html` | Оверлей-виджет для стрима (добавить в OBS как Browser Source) |

## Запуск локально

```bash
npm install
npm start
```

Откройте: http://localhost:3000/admin.html

## Деплой на Render.com

1. Загрузи проект на GitHub
2. На [render.com](https://render.com) → New → Web Service → подключи репозиторий
3. Render сам подхватит `render.yaml` с настройками
4. После деплоя:
   - Админка: `https://ваш-домен.onrender.com/admin.html`
   - Виджет: `https://ваш-домен.onrender.com/widget.html`

## Использование в OBS

1. Добавь `Browser Source`
2. URL: `https://ваш-домен.onrender.com/widget.html`
3. Ширина: `400`, Высота: `600`
4. Включи: **Shutdown source when not visible** / **Refresh browser when scene becomes active**

## Как работает

### Система иксов

| Показатель | Формула | Описание |
|-----------|---------|----------|
| **Бонус X** | сумма(результатов) / сумма(стоимостей) | Окупаемость конкретно бонусок |
| **Общий X** | текущий_баланс / стартовый_баланс | Общий результат всей сессии |
| **Множитель бонуски** | результат / цена | Колонка рядом с каждой бонуской |

### Workflow для стримера

1. В начале сессии: вписать **стартовый баланс** (кнопка ↺ рядом с «Обновить»)
2. Перед каждой бонуской: **Добавить бонуску** (название, цена)
3. После открытия: вписать **результат** в строку с активной бонуской → Enter
4. Следующая бонуска автоматически выделяется в виджете
5. Периодически обновлять текущий баланс кнопкой «Обновить»
